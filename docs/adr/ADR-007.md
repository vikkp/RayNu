# ADR-007: VMware Migration as Dedicated Workstream

- **Status:** Accepted
- **Date:** 2026-06-01
- **Pillars:** [A] [Z]

## Context

Importing VMs from vCenter (VMDK/OVF) is a commercial differentiator but is large, dependency-heavy, and not security-isolation-critical in the same way as EPT/VMCS. Folding it into the Proven Core or the critical path of M4/M5 would inflate risk R11.

## Decision

Treat VMware migration as a **dedicated workstream** with its own milestone **M5.5** (parallel to late M5 / early M6). The `migrate/` subsystem is **outside** the Proven Core.

Gate: 10+ VMs migrated from vCenter in one command.

## Consequences

- Migration code must not pull new modules into the Proven Core without a separate ADR.
- Audit events for migrate start/complete/fail are still mandatory ([A]).
- Binary size impact of migration assets follows ADR-003 (lazy load).

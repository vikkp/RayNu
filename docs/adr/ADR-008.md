# ADR-008: Proof Maintenance & Toolchain Pinning

- **Status:** Accepted
- **Date:** 2026-06-01
- **Pillars:** [V]

## Context

Verus and Rust nightly move quickly. Unpinned toolchains cause silent proof bitrot (R12, R14). After L3 delivery, maintenance must be budgeted explicitly.

## Decision

- Pin Rust nightly in `rust-toolchain.toml`.
- Pin Verus in `verus-version.toml`.
- Re-verify proofs on every commit against **pinned** versions.
- Run a **nightly** CI job against latest Verus to detect upcoming breakage.
- Treat toolchain upgrades as discrete engineering tasks with their own time budget.
- Budget **~1 week/quarter** for proof maintenance after initial L3 delivery.

## Consequences

- Developers use the pinned toolchain; floating `nightly` in local overrides is discouraged for Proven Core work.
- Soft-fail Verus/Kani CI is acceptable until the pinned toolchain is installed in CI images.
- Upgrade PRs must re-run the full Proven Core verify and document breakage.

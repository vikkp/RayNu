# ADR-003: Single-Binary Asset Strategy

- **Status:** Accepted
- **Date:** 2026-06-01
- **Pillars:** [Z]

## Context

RayNu-V competes on zero-config simplicity: one `.efi`, no OS, no packages, no external config files. Guest kernels, Web UI, schemas, and templates must still ship somehow without destroying boot time or blowing the size budget.

## Decision

Embed assets as PE/COFF sections on the single `r640-hypervisor.efi` binary:

- `.assets.kernel`, `.assets.initrd`, `.assets.webui`, `.assets.schemas`, `.assets.vmconfigs`
- Compress with zstd; **lazy-decompress at first use** for non-critical assets
- **Target:** 15 MB total · **Hard limit:** 20 MB (triggers size audit)
- **Fallback:** split-mode (`.efi` core + `/assets/` on ESP) for constrained environments

## Rationale

- Preserves the [Z] story: copy one file, boot, done.
- Lazy load keeps M0/M1 boot path fast.
- Size budget makes regressions visible via `tools/check-size.sh`.

## Consequences

- Asset additions must update the size budget and CI size check.
- Build pipeline embeds kernel/initrd via `#[link_section]` (M3.22); `tools/check-pe-assets.sh`
  verifies `.askern` / `.asinit` / `.aswebui` / `.aschema` (8-char COFF aliases for
  `.assets.kernel` / `.assets.initrd` / `.assets.webui` / `.assets.schemas`).
- Boot prefers PE embed; ESP `\EFI\BOOT\*` is the split-mode fallback.
- Web UI SPA embedded in M5.2 (`mgmt/webui.rs`, marker `RAYNU-V-M5-WEBUI-OK`); report
  schemas embedded in M5.4 (`audit/report.rs`, marker `RAYNU-V-M5-REPORT-OK`).
  zstd for large assets remains deferred while payloads stay tiny under the 15 MB target.
- Marker: `RAYNU-V-M3-ASSETS-OK` (kernel/initrd) · `RAYNU-V-M5-WEBUI-OK` · `RAYNU-V-M5-REPORT-OK`.
